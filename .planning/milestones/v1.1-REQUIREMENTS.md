## v1.1 Requirements (All Completed)

### Resiliência

- [x] **RESIL-01**: Admin pode acionar recálculo manual de risco imediatamente após importação CSV
  - **Outcome:** Auto-trigger implementado em `confirmar()` — encadeia calcularRiscos() + checkAndSendAlerts() automaticamente. Botão manual removido do UI.
- [x] **RESIL-02**: Sistema retenta fetch Open-Meteo com backoff exponencial em caso de falha
  - **Outcome:** fetchWithRetry() atualizado para exponential backoff (5s → 10s → throw). Stale cache servido durante retentativas.
- [x] **RESIL-03**: Usuário vê banner de dados stale quando forecast está desatualizado
  - **Outcome:** Banner #banner-stale-forecast exibido quando last_fetched_at > 120min. Verificado em smoke test (09-03).

### UX

- [x] **UX-01**: Usuário pode configurar antecedência de alerta (1h a 48h) via seletor
  - **Outcome:** Seletor #sel-alert-hours já estava funcional desde v1.0 (campo alert_threshold no banco). Verificado em smoke test (09-03). Nenhum código novo necessário.
- [x] **UX-02**: Usuário vê painel de status do sistema (freshness do forecast, calendar sync, push ativo)
  - **Outcome:** Aba Status com 3 indicadores em tempo real implementada em 09-01. Visível apenas para usuários autenticados.
- [x] **UX-03**: Usuário vê histórico de alertas recebidos
  - **Outcome:** GET /api/alertas/historico com requireAuth e paginação. Tab Alertas com cards em pt-BR implementada em 09-02.
- [x] **UX-04**: Usuário novo é guiado por wizard de 3 passos (login → calendar → push) no primeiro acesso
  - **Outcome:** Wizard modal completo implementado em Phase 10. Flag onboarding_done persistido no banco. Verificado em checkpoint humano.
