    ### Verificar RESIL-03: Banner de dados stale

    **Cenário A — Forecast atualizado (comportamento normal):**
    1. Abrir http://localhost:3000
    2. Confirmar que o banner "#banner-stale-forecast" NÃO está visível (o forecast provavelmente está recente)
    3. Abrir DevTools → Network → filtrar por "atual" → confirmar que a resposta JSON NÃO contém `"stale": true`

    **Cenário B — Simular forecast stale (forçar o banner):**
    Opção 1 — Via SQLite: Conectar ao banco e executar:
    ```sql
    UPDATE previsao_meta SET last_fetched_at = datetime('now', '-3 hours');
    ```
    Depois recarregar a página e confirmar que o banner aparece com texto de aviso.
    Reverter: `UPDATE previsao_meta SET last_fetched_at = datetime('now');`

    Opção 2 — Se não tiver acesso direto ao SQLite: No DevTools → Network, interceptar a resposta de `/api/previsao/atual` e confirmar que quando `stale: true` está no JSON, o banner fica visível.

    **Critério de aprovação RESIL-03:** Banner visível quando stale=true; invisível quando stale=false.

    ---

    ### Verificar UX-01: Seletor de antecedência

    **Pré-requisito:** Usuário autenticado com push notifications ATIVO.

    1. Fazer login com Google
    2. Na aba "Calendário", ativar push notifications (se ainda não ativo)
    3. Confirmar que o seletor de antecedência aparece na seção de push (horas antes do evento)
    4. O seletor deve ter as opções: 1h, 2h, 6h, 12h, 24h, 48h
    5. Alterar o valor do seletor para uma opção diferente da atual
    6. Confirmar que aparece a mensagem "Antecedência atualizada!" (texto definido em main.js linha 988)
    7. Abrir DevTools → Network → confirmar que houve um PATCH para `/api/push/alert-hours` com status 200
    8. Recarregar a página, fazer login novamente e confirmar que o seletor mantém o valor escolhido (persistência)

    **Cenário adicional — Push inativo:**
    9. Desativar push (clicar "Desativar notificações")
    10. Confirmar que o seletor de antecedência NÃO é visível quando push está inativo

    **Critério de aprovação UX-01:** Seletor visível com push ativo; PATCH bem-sucedido; valor persiste após reload; seletor oculto com push inativo.
