<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → POST /api/admin/confirmar | Attacker could craft a confirmar request with arbitrary linhas payload, bypassing the preview step entirely |
| linhas[].nivel value → NIVEIS_VALIDOS check | A crafted confirmar payload could contain any string for nivel |
| linhas[].data value → INSERT criado_em | A crafted payload could attempt to set criado_em to a future date or inject SQL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-09 | Tampering | AdminController.confirmar — trusting preview payload | mitigate | Controller re-runs NIVEIS_VALIDOS allowlist check and required field presence check on every linhas entry before insert. The preview round-trip is for UX only — confirmar is hardened independently. |
| T-07-10 | Tampering | SQL injection via linhas[] field values in INSERT | mitigate | INSERT uses ? placeholders for all six field values. No string concatenation in any db.run call. |
| T-07-11 | Tampering | Historical date manipulation (criado_em set to future) | accept | Risk engine uses criado_em only to count historical occurrences for risk scoring. A future-dated row inflates one bairro's risk score. Acceptable for v1 single-operator system — any authenticated user is already trusted. |
| T-07-12 | Elevation of Privilege | Unauthenticated confirmar call causing bulk insert | mitigate | requireAuth on POST /api/admin/confirmar; returns 401 if req.session.userId absent. |
| T-07-13 | Information Disclosure | erros array in confirmar response leaking DB state | accept | erros contains only the input dados object and a generic error string. No row IDs, no schema details leaked. Low sensitivity for v1. |
</threat_model>
