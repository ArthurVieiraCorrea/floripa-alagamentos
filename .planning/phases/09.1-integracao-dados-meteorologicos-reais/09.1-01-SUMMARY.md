## What Was Built

### Task 1: DDL weather_observations

Adicionado ao final do bloco principal de `CREATE TABLE/INDEX` em `initSchema()`, imediatamente antes das migrações `ALTER TABLE`:

```sql
CREATE TABLE IF NOT EXISTS weather_observations (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  bairro          TEXT    NOT NULL DEFAULT 'florianopolis',
  observed_time   TEXT    NOT NULL,
  precipitacao_mm REAL    NOT NULL DEFAULT 0,
  fonte           TEXT    NOT NULL,
  fetched_at      TEXT    NOT NULL DEFAULT (datetime('now')),
  UNIQUE(bairro, observed_time, fonte)
)

CREATE INDEX IF NOT EXISTS idx_weather_obs_bairro_time
  ON weather_observations(bairro, observed_time)
```

- Constraint `UNIQUE(bairro, observed_time, fonte)` permite que Visual Crossing e REDEMET registrem dados para o mesmo timestamp sem conflito
- `CREATE TABLE IF NOT EXISTS` e `CREATE INDEX IF NOT EXISTS` garantem idempotência em bancos existentes
- Tabelas `forecasts`, `forecasts_meta`, `risk_scores`, `usuarios`, `alertas_enviados` e demais permanecem intactas
