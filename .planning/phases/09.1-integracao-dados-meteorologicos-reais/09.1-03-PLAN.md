REDEMET API endpoint (confirmado via curl — endpoint ativo):
```
GET https://api-redemet.decea.mil.br/mensagens/metar/SBFL?api_key={KEY}&data_ini={YYYYMMDDHH}&data_fim={YYYYMMDDHH}
```
- YYYYMMDDHH = hora UTC atual (ex: "2026042314")
- Sem data_ini/data_fim retorna apenas o ultimo METAR (menos robusto)
- api_key como query param (nao Authorization header)

Estrutura da resposta JSON do REDEMET (confirmada via documentacao DECEA):
```json
{
  "status": true,
  "message": 200,
  "data": {
    "current_page": 1,
    "data": [
      {
        "id_localidade": "SBFL",
        "validade_inicial": "2026-04-23 14:00:00",
        "mens": "METAR SBFL 231400Z 12008KT 9999 -RA FEW025 SCT060 22/19 Q1015=",
        "recebimento": "2026-04-23 13:57:00"
      }
    ]
  }
}
```
Array alvo: data.data.data[] (duplamente aninhado)
Campos relevantes: mens (string METAR), validade_inicial (timestamp UTC)

Descoberta critica (RESEARCH.md Pitfall 1):
METARs brasileiros NAO incluem P-group (P0015 etc.) — e adicao exclusiva de estacoes NWS/FAA norte-americanas.
Estrategia: precipitacao_mm = 0 SEMPRE para fonte redemet-metar.

**buildUrl() — hora UTC como YYYYMMDDHH:**
```javascript
function buildUrl() {
  const KEY = process.env.REDEMET_API_KEY;
  const nowUtc = new Date();
  const pad = n => String(n).padStart(2, '0');
  const dataHora = [
    nowUtc.getUTCFullYear(),
    pad(nowUtc.getUTCMonth() + 1),
    pad(nowUtc.getUTCDate()),
    pad(nowUtc.getUTCHours())
  ].join(''); // ex: "2026042314"
  return `https://api-redemet.decea.mil.br/mensagens/metar/SBFL?api_key=${KEY}&data_ini=${dataHora}&data_fim=${dataHora}`;
}
```

**fetchWithRetry(url) — copiar logica do forecastService com url como parametro:**

- Log prefix [redemet] em todos os console.warn/error
- Throw lastErr apos esgotar tentativas

    // precipitacao_mm = 0 SEMPRE para fonte redemet-metar
    db.run(

} catch (dbErr) {
  try { db.run('ROLLBACK'); } catch (_) {}

  return;
}
console.log(`[redemet] ${registros.length} METAR(s) processados para SBFL`);

| Boundary | Description |
|----------|-------------|
| REDEMET API → Node.js fetch | Resposta JSON de API externa parseada e persistida no SQLite |
| process.env → buildUrl() | API key lida de variavel de ambiente e inserida como query param na URL — nunca logada |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09.1-03-01 | Information Disclosure | REDEMET_API_KEY na URL do fetch | mitigate | Chave aparece apenas na URL construida internamente em buildUrl(); nunca logada via console.*; URL nao e armazenada no banco |
| T-09.1-03-02 | Tampering | Campo mens do METAR injetado via SQL | mitigate | O campo mens e usado apenas em regex match e console.log — nunca inserido no banco via parametros SQL; INSERT usa apenas [observed_time] como parametro (precipitacao_mm = 0 literal) |
| T-09.1-03-03 | Tampering | validade_inicial malformado causando data invalida | mitigate | new Date() com string invalida retorna Invalid Date; .getTime() retorna NaN; NaN - offset = NaN; new Date(NaN).toISOString() lanca erro que e capturado pelo try/catch da transacao com ROLLBACK |
| T-09.1-03-04 | Denial of Service | REDEMET API indisponivel ou retornando erro | accept | fetchWithRetry loga e retorna sem throw (D-10); sistema continua sem dados REDEMET; Visual Crossing e Open-Meteo nao sao afetados |
| T-09.1-03-05 | Spoofing | Endpoint REDEMET substituido por resposta falsa | accept | Conexao HTTPS com TLS verification padrao do Node.js; chave falsa retornaria status=false que e checado antes do processamento |
