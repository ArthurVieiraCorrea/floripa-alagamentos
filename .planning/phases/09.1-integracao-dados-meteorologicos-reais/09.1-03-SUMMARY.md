# Summary: Plan 09.1-03 — redemetService.js

## What Was Built

`backend/src/services/redemetService.js` — serviço que busca o METAR horário do SBFL (aeroporto de Florianópolis) via REDEMET API e persiste em `weather_observations` com `precipitacao_mm=0` sempre (METARs brasileiros não fornecem P-group quantitativo). O valor é qualitativo: confirmar presença/ausência de precipitação e logar o tipo (RA, TSRA, DZ).

## Key Files

### Created
- `backend/src/services/redemetService.js` — 161 linhas; exporta `fetchAndCacheObservations()`

## Implementation Details

- **URL:** `https://api-redemet.decea.mil.br/mensagens/metar/SBFL?api_key=...&data_ini=YYYYMMDDHH&data_fim=YYYYMMDDHH`
- **Timestamp:** Hora UTC atual formatada como YYYYMMDDHH para `data_ini` e `data_fim`
- **Retry:** 3 tentativas com backoff exponencial via `AbortController`
- **Guard D-09:** Retorna silenciosamente se `REDEMET_API_KEY` ausente
- **No-throw D-10:** Erros logados, sem re-throw
- **precipitacao_mm = 0 literal:** METARs brasileiros não incluem P-group (ICAO Annex 3)
- **UTC → UTC-3:** `validade_inicial` convertido com `getTime() - 3 * 60 * 60 * 1000`
- **Parsing qualitativo:** regex `/\s(-|\+)?(VC)?(TS|SH|FZ)?(DZ|RA|SN|SG|IC|PL|GR|GS)/` para grupos de tempo presente
- **NAO atualiza `forecasts_meta`** — responsabilidade exclusiva do `visualCrossingService`
